codigo P

En la asignatura de Lenguajes y Autómatas, el "código P" se refiere históricamente a un código de ensamblador objetivo estándar producido por los compiladores Pascal en las décadas de 1970 y 1980, diseñado como código intermedio para una máquina hipotética de pila (la máquina P) con el fin de facilitar la portabilidad de los compiladores. Este código intermedio se caracterizaba por ser explícito en la descripción de su ambiente de ejecución, incluyendo tamaños de datos y otra información específica de la máquina P.

Conceptos

¿Qué es el Código Intermedio?

01

El código intermedio es una representación abstracta del programa fuente que se genera después del análisis sintáctico y antes de la generación de código máquina. Sirve como puente entre el lenguaje de alto nivel y el lenguaje ensamblador o máquina

Código de Pila (Stack Code)

02

- Utiliza una pila para almacenar operandos y resultados.PUSH a PUSH b ADD PUSH c MUL

Árboles de Sintaxis Abstracta (AST)

03

Representa la estructura jerárquica del código fuente, con nodos que simbolizan expresiones y operaciones. Es una representación en forma de árbol muy utilizada para la optimización.

Propiedades del Código Intermedio

04

Independencia de la Máquina: Es una de sus propiedades más importantes, ya que permite que el mismo código intermedio sea traducido a diferentes arquitecturas de máquina. Facilidad de Producción y Optimización.
